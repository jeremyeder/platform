Alt UI mockups

Three mockups of the proposed alternative frontend for the Ambient Code Platform. A complete rebuild in a distinct design system: dark, dense, command-center aesthetic. Deliberately different from the existing NextJS + Shadcn light frontend in palette, typography, density, and navigation shape.

01
Projects
Landing view: persistent left rail, breadcrumb + ⌘K command palette, stats strip, filter chips, data table with hover-reveal row actions.
02
Sessions list
A single project's sessions, with project sub-nav (Sessions / Scheduled / Keys / Permissions / Settings) and phase filter chips with live counts.
03
Session detail
The busiest page — chat stream plus file explorer, with a single unified file toolbar (breadcrumb + size + actions on one row). Fixes the nested-download-buttons pain point from the current UI.

Notifications & context window

The alt UI treats notifications and context window pressure as first-class concerns, since both are load-bearing for an agent platform.

  • Browser tab dot. The favicon is an SVG with a red notification circle overlaid on the Ambient mark. When the count drops to zero the overlay disappears — this tab is the "is anything waiting for me" signal.
  • Topbar bell. A bell icon with a badge count sits next to ⌘K in the topbar on every page. Opens a panel (not shown in static mockups) with session notifications (AskUserQuestion pending, rate limit recovery, session failed, compaction happened, MCP connected).
  • Session row pulse. On the sessions list, any row with pending notifications shows a pulsing red dot next to its title and a Waiting for input phase — see sess-q7mv22 on mockup 2.
  • Context window meter. On the session detail topstrip, a dedicated widget shows % + tokens / max + compaction count. Color ramps from green → amber → red as usage climbs. 2× next to the stack icon means compaction has fired twice so far in this session.
  • Inline system events. The chat stream interleaves first-class system events (not normal chat messages) with a pill + rule line: Context compacted · 38 older messages summarized · 94K → 52K tokens, Rate limit hit · retrying in 28s, etc. This gives an audit trail of what happened around the agent's work without breaking the message flow.

Responsive behavior

All three mockups are fully responsive. Resize the browser below ~768px (or open in devtools device mode) to see the mobile layout.

  • Phone (≤768px). Left rail becomes a hamburger drawer with backdrop. Topbar collapses ⌘K to a search icon and shows only the current breadcrumb segment. Stats strip becomes a 2×2 grid. Filter chips and project sub-nav scroll horizontally. Data tables collapse to stacked cards with CSS-driven field labels and always-visible row actions.
  • Small phone (≤400px). Stats strip stacks to a single column. Breadcrumbs truncate to ~52vw.
  • Tablet (≤1100px). Rail narrows, content padding tightens, explorer panel shrinks but stays side-by-side with chat.
  • Session detail on phone. The chat + explorer split becomes a segmented control (Chat / Files). One pane is visible at a time. The file viewer toolbar still uses a single row — breadcrumb segments truncate, actions stay fixed.
  • Touch targets. All icon buttons bump from 28×28 to 36×36 on phone. Filter chips grow to 30px. Composer send button grows to 36px.

Design language

Palette. Dark surfaces (#0a0c10 → #262b35) with cool grays. Single accent color — violet-indigo #7c5cff. Status as colored dots, not pills: running, idle, error, stopped, pending.

Type. Inter for UI, JetBrains Mono for IDs, paths, timestamps, numbers. 13px base. Tight -0.005em tracking.

Shape. 4px radii (not pill-rounded). 1px hairlines instead of shadows. Square icon buttons (28×28). Tables over cards for lists.

Navigation. Persistent left rail (always visible) with top-level destinations and workspace-scoped items below a divider. Breadcrumb in the topbar for drill-down. ⌘K palette is the fast path for power users.

What this fixes vs. the current UI

  • No nested toolbars. The file viewer has exactly one toolbar row. Name, type, size, and download all live together. No more stacked Download buttons.
  • No "bolted together" action slots. Every trailing-actions region is a single icon-btn group with consistent sizing and spacing, not a grab-bag of ghost buttons at different weights.
  • One breadcrumb, not two. File location and navigation share the same breadcrumb component — clickable segments, with root dir as a dropdown.
  • Status at a glance. Colored dots (with subtle ring) replace verbose badges. Faster to scan 40 rows for "what's broken".
  • Stats where you need them. Projects page stat strip answers "what's happening right now" before you pick a project to open.

Scope & next steps

These mockups are static HTML with no JavaScript — pure design artifacts under components/frontend/mockups/. Nothing under components/frontend/src/ has been touched.

Once you approve the direction, the plan is to build a new package at components/frontend-console/ (Vite + React + a distinct component layer), with full feature parity against the existing frontend, gated behind an env var / Unleash flag so operators can opt their deployment into the new UI. Auth header forwarding and the /api/* proxy pattern will mirror the existing Next.js app.
